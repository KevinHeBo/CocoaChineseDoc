Introduction

Overview

Xcode is a mature and advanced suite of developer tools used to create software products. This suite includes GUI (graphical user interface)–based applications, command-line tools, and documentation to assist you in the software development process. The centerpiece of this suite is the Xcode application, which provides an elegant and powerful user interface for creating and managing software development projects. You use Xcode to organize and edit your source files, view documentation, build your application, debug your code, and optimize your application’s performance.

To develop iPhone applications, you use the following Xcode tools.

  • Interface Builder. The Interface Builder application lets you design your application’s user interface graphically and save those designs as resource files that your application loads at runtime.

  • iOS Simulator. The iOS Simulator application implements the iOS API, providing an environment that closely resembles the environment iOS devices provide. It allows you to run your applications in Mac OS X, letting you test most of their functionality without the need for an iOS device.

  • Instruments. The Instruments application is a powerful performance measurement tool that lets you peer into your code as it’s running and gather important metrics about what it is doing. You can view and analyze the data Instruments collects in real time, or you can save that data and analyze it later.

  • Shark. Shark is a code-profiling application that lets you analyze the performance of your code and allows you to profile the entire system. Shark profiles the system while your code is running to see where time is being spent and can also produce profiles of performance events. This information is an invaluable first step in your performance tuning effort so you can see which parts of your code or the system are the bottlenecks.

These are only the essential tools you use to develop iPhone applications. Xcode uses industry-standard and open-source tools, such as GCC and GDB, to compile and debug your code. You can use these and other command-line tools directly if you prefer a traditional UNIX-based development environment. For team-based development, Xcode also interacts seamlessly with source control systems, making it easy for individuals to safeguard their work and keep track of their team’s progress.

For developers writing web applications targeting iOS devices, Dashcode takes advantage of some iOS SDK features to make building these apps even easier. To learn more, see Dashcode User Guide.

The Basics

Before you begin writing code:

  • Read iOS Technology Overview to get a high-level introduction to iOS. Also read iOS Development Guide for an introduction to the environment Xcode provides for developing iPhone applications.

  • Read The Xcode Editor in Xcode Workspace Guide to learn about the advanced features you can take advantage of while writing and debugging your code. Read “Documentation Access” in the same document to find out how to locate API reference or high-level documentation when you need it.

  • Read Xcode Debugging Guide to learn about the debugging approaches you can take to squash bugs in your code.

  • Read Instruments User Guide to learn how to measure the performance of your applications and ensure that they use the resources available in iOS devices as efficiently as possible, a critical step to providing a compelling user experience to your customers.
