Introduction

Overview

Virtually all iPhone applications need to manage data. Data management involves the creation and handling of the various types of data available to a program, including strings, text, binary data, dates, collections, property lists, and XML data. iPhone applications use data management interfaces to store and access data in local databases, files, folders, and bundles. They also use data management interfaces to receive and respond appropriately to events and other types of messages, such as notifications.

iOS data management interfaces reside in several different frameworks. The principal frameworks are:

  • UIKit—An Objective-C framework that provides the key infrastructure for implementing graphical, event-driven applications in iOS. UIKit includes such classes as UIResponder, UIApplication, and UIEvent to manage user interface data.

  • Foundation—An Objective-C framework that defines a base layer of classes that can be used for any type of iPhone program. Foundation classes provide object wrappers or equivalents for primitives such as numeric values, strings, and collections. Foundation also includes utility classes for accessing underlying system entities and services, such as ports, threads, and file systems.

  • Core Data—A generalized framework for object graph management and persistence. Core Data includes support for features such as undo and redo, change propagation—including maintaining the consistency of relationships among objects, and storing objects in external data repositories.

  • Core Foundation—A set of programming interfaces conceptually derived from the Foundation framework but implemented in the C language. Core Foundation defines opaque types for data objects such as strings, collections, time and dates, ports, application run loops, interprocess communication, and preferences.

The Basics

Before you begin writing code:

  • Read The Core Application Design in iOS Application Programming Guide to learn how to design an iPhone application, or The Core Application Design in iPad Programming Guide to learn how to design an iPad application.

  • Watch the video iPhone Application Development - Getting Started to learn about the iOS programming interfaces and how to use them.

Next Steps

To support data management in your software, you need to decide which frameworks to use for your particular needs and experience. If your software has a user interface, you probably need to handle Multi-Touch events. You may want to provide application-level preferences. Your software may need to know the orientation of the device (accelerometer data) or the location of the device (location data).

Deciding Which Frameworks to Use

Several different iOS frameworks provide interfaces to help developers manage data in their applications.

To get started:

  • If you’re new to Objective-C, start with Foundation Framework Reference to learn about the classes in the Foundation framework. The framework includes classes for working with data such as arrays, dictionaries, strings, dates, and timers. It also includes classes for interacting with the file system.

  • If you’re a Cocoa developer, start with UIKit Framework Reference to learn about the classes in the UIKit framework. The framework includes classes for working with data such as acceleration, device, screen, event, responder, and touch.

  • If you already have experience with iPhone development, to learn how to use Core Data for general object graph management follow the path outlined in Getting Started with Core Data.

  • If you want to explore a C interface for basic data management, start with Core Foundation Framework Reference. The framework includes opaque types for working with data such as arrays, dictionaries, strings, bundles, and file descriptors.

  • To learn how to determine the current latitude and longitude of a device, start with Core Location Framework Reference.

  • If you need access to the contacts on the device, start with Address Book Framework Reference for iOS. The Address Book framework allows you to retrieve and add your own properties to the people and groups in the contacts database.

Handling Multi-Touch Events

In iOS, a touch is the presence or movement of a finger on the screen that is part of a unique Multi-Touch sequence. If your program has a user interface, it probably needs to handle Multi-Touch events.

To get started:

  • Read Event Handling in iOS Application Programming Guide for an introduction to events in iOS, which are based on the Multi-Touch interface model.

  • See the sample project TheElements, which handles Multi-Touch zooming and scrolling.

Providing Application-Level Preferences

Preferences are settings used to configure the behavior or appearance of an application. Users of iOS devices expect applications to present their settings in a consistent manner.

To get started:

  • Read Application Preferences in iOS Application Programming Guide to learn how to display application-level preferences using the system-supplied Settings application.

  • See the sample code project Metronome, a simple representation of a swinging-arm mechanical metronome. This project provides an example of how to store application preferences.

  • If you need more flexibility than that offered by the Settings application, you can manage preferences within your application using the NSUserDefaults class in the Foundation framework. For more information, see User Defaults Programming Topics.

Accessing Accelerometer Events

An accelerometer measures changes in velocity along a given linear path. iOS devices contain three accelerometers, one along each of the primary axes of the device. If you’re interested in receiving the accelerometer data and using it as input to your application, you can do so using the UIAccelerometer class in the UIKit framework.

To get started:

  • Read “Accessing Accelerometer Events” in iOS Application Programming Guide to learn how to configure and receive accelerometer events.

  • See the sample code project BubbleLevel, which uses accelerometer data to create a visual bubble level and inclinometer. This project provides an example of how to detect device orientation.

Getting the Device Location

iOS devices contain hardware that can determine the device’s current location, triangulating a position fix from available signal information. You can use the CLLocation class in the Core Location framework to get the altitude, the altitude's vertical accuracy, and the geographical coordinate for the device.

To get started:

  • Read Core Location Framework Reference. You use the classes and protocols in this framework to configure and schedule the delivery of location-related events.

  • Read “Getting the User's Current Location” in iOS Application Programming Guide to learn how to initiate and process location updates.

Accessing Contact Information

Contacts is a centralized database for contact and other personal information for people. Contact information is important for software such as email and chat programs.

To get started:

  • Read Address Book Programming Guide for iOS to learn how to leverage the contacts database in your application. You can not only access a user’s contact data but also design and implement your own properties and actions for the data.
